# 网页版淘宝用户侧电商应用需求文档

## 一、项目概述

构建一个网页版淘宝用户侧电商应用，为用户提供完整的购物体验，包括商品浏览、搜索、购买、订单管理等核心功能。

## 二、核心功能模块

### 2.1 用户认证模块
- 用户注册（手机号/邮箱注册）
- 用户登录（账号密码登录、短信验证码登录）
- 密码找回
- 用户信息管理（昵称、头像、个人资料）
- 收货地址管理

### 2.2 商品浏览模块
- 首页展示
  - 轮播广告位
  - 商品分类导航
  - 热门商品推荐
  - 限时优惠活动
- 商品搜索
  - 关键词搜索
  - 分类筛选
  - 价格区间筛选
  - 销量/价格排序
- 商品列表展示
  - 商品图片、名称、价格、销量
  - 网格/列表视图切换
  - 分页加载

### 2.3 商品详情模块
- 商品基本信息展示
  - 商品图片轮播
  - 商品标题、价格、销量
  - 商品规格选择（颜色、尺寸等）
  - 商品库存显示
- 商品详情页
  - 图文详情展示
  - 商品参数说明
- 商品评价展示
  - 评价列表
  - 评价筛选（好评/中评/差评）
  - 晒图评价

### 2.4 购物车模块
- 添加商品到购物车
- 购物车商品列表展示
- 商品数量修改
- 商品删除
- 商品选中/全选
- 价格计算（小计、总价）
- 结算跳转

### 2.5 订单模块
- 订单创建
  - 选择收货地址
  - 选择商品规格
  - 备注填写
  - 优惠券使用（可选）
- 订单支付（模拟支付流程）
- 订单列表
  - 全部订单
  - 待付款
  - 待发货
  - 待收货
  - 待评价
- 订单详情查看
- 订单取消
- 确认收货
- 订单评价

### 2.6 用户中心模块
- 个人信息展示与编辑
- 收货地址管理
- 我的收藏
- 浏览历史
- 我的订单入口

## 三、Use Case 用例场景

### 场景 1: 新用户注册并完成首次购物
1. 用户访问网站首页，点击"注册"
2. 填写手机号、密码完成注册
3. 浏览商品分类或搜索商品
4. 查看商品详情，选择规格
5. 加入购物车
6. 进入购物车，勾选商品
7. 提交订单，填写收货地址
8. 完成支付（模拟）
9. 查看订单状态

### 场景 2: 老用户快速购物
1. 用户登录账号
2. 在搜索框输入关键词搜索商品
3. 使用筛选条件过滤商品
4. 快速浏览商品列表
5. 点击商品查看详情
6. 直接"立即购买"
7. 确认订单信息并支付

### 场景 3: 购物车管理
1. 用户登录后查看购物车
2. 修改商品数量
3. 删除不需要的商品
4. 选择部分商品结算
5. 查看价格变化

### 场景 4: 订单跟踪与评价
1. 用户登录后进入"我的订单"
2. 查看待发货订单状态
3. 查看待收货订单，确认收货
4. 对已收货订单进行评价
5. 查看已完成订单历史

### 场景 5: 收藏商品对比购买
1. 用户浏览多个商品
2. 将感兴趣的商品加入收藏
3. 在收藏夹中对比价格和规格
4. 选择合适的商品加入购物车
5. 完成购买

## 四、非功能性需求

### 4.1 性能需求
- 页面加载时间 < 2秒
- API 响应时间 < 500ms
- 支持 1000+ 并发用户
- 图片懒加载优化

### 4.2 可用性需求
- 响应式设计，支持桌面和移动端
- 界面简洁易用，符合用户习惯
- 关键操作有明确的反馈提示
- 表单输入有实时验证

### 4.3 安全性需求
- 用户密码加密存储（bcrypt）
- JWT Token 身份认证
- HTTPS 传输加密
- SQL 注入防护
- XSS 攻击防护
- CORS 跨域配置

### 4.4 可维护性需求
- 前后端分离架构
- 代码模块化、组件化
- 统一的错误处理机制
- 完善的日志记录
- API 文档自动生成

### 4.5 可扩展性需求
- 微服务架构预留
- 数据库读写分离预留
- 缓存机制支持
- 文件存储可切换（本地/云存储）

## 五、技术选型

### 5.1 前端技术栈
- **框架**: React 18 + TypeScript
- **状态管理**: Zustand / React Context
- **路由**: React Router v6
- **UI 组件库**: Ant Design / Tailwind CSS
- **HTTP 客户端**: Axios
- **构建工具**: Vite

### 5.2 后端技术栈
- **语言**: Go 1.21+
- **Web 框架**: Gin
- **ORM**: GORM
- **数据库**: MySQL 8.0+
- **缓存**: Redis (可选)
- **认证**: JWT

### 5.3 开发工具
- **版本控制**: Git
- **API 文档**: Swagger/OpenAPI
- **代码规范**: ESLint + Prettier (前端), golangci-lint (后端)

## 六、数据库主要实体

### 用户 (users)
- ID, 用户名, 密码, 手机号, 邮箱, 头像, 创建时间, 更新时间

### 商品分类 (categories)
- ID, 名称, 父分类ID, 图标, 排序

### 商品 (products)
- ID, 分类ID, 名称, 描述, 价格, 库存, 销量, 主图, 图片列表, 状态, 创建时间

### 商品规格 (product_specs)
- ID, 商品ID, 规格名称, 规格值, 库存, 价格

### 购物车 (carts)
- ID, 用户ID, 商品ID, 规格ID, 数量, 选中状态

### 订单 (orders)
- ID, 用户ID, 订单号, 总金额, 支付金额, 状态, 收货地址, 创建时间, 支付时间

### 订单项 (order_items)
- ID, 订单ID, 商品ID, 规格ID, 商品名称, 商品图片, 价格, 数量

### 收货地址 (addresses)
- ID, 用户ID, 收货人, 手机号, 省市区, 详细地址, 是否默认

### 商品评价 (reviews)
- ID, 用户ID, 订单ID, 商品ID, 评分, 内容, 图片, 创建时间
